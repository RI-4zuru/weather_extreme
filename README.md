# weather_extreme

気象庁の公開データをもとに、都道府県ごとの県内極値一覧と実況値を組み合わせて表示する GitHub Pages 用のウェブアプリです。

各観測所の極値順位を一覧で確認できるほか、最新の実況値から「県内極値の上位に入りそうな値」をまとめて確認できることを目指しています。

現在は機能拡張により、**地域単位・全国単位での極値ランキング表示にも対応**しています。

---

## このページでできること

* 都道府県・月を切り替えて県内極値一覧を表示
* 地域単位・全国単位での極値ランキング表示
* 要素ごとに、各観測所の 1 位〜10 位の記録を表示
* 観測所ごとに観測開始日を併記
* 実況値をもとに、ランキングへ自動挿入
* 「ランクインあり」「極値1位更新あり」の判定表示
* 「実況で10位以内に入った項目」の一覧表示とジャンプ機能
* 直近の記録や実況ランクインを色分け表示
* GitHub Actions によるデータ更新

---

## 主な表示要素

### 降水

* 日降水量
* 日最大10分間降水量
* 日最大1時間降水量
* 月最大3/6/12/24/48/72時間降水量
* 月降水量（多い方 / 少ない方）

### 気温

* 日最高気温（高い方 / 低い方）
* 日最低気温（高い方 / 低い方）
* 月平均気温（高い方 / 低い方）

### 湿度

* 日最小相対湿度

### 風

* 日最大風速
* 日最大瞬間風速

### 日照

* 月間日照時間（多い方 / 少ない方）

### 雪

* 降雪の深さ日合計
* 降雪の深さ月合計
* 月最大3/6/12/24/48/72時間降雪量
* 月最深積雪（大きい方 / 小さい方）

## ページの見方

### 1. 都道府県・地域・全国の切り替え

画面上部で以下を選択します。

* 都道府県
* 地域（例：近畿・関東甲信など）
* 全国

地域・全国を選択した場合は、以下の構成で表示されます。

```text
地域（または全国）
地域総合（または全国総合）
都道府県総合
```

---

### 2. 要素の切り替え

要素選択から、表示したい気象要素を切り替えます。

* 一度に表示する要素は 1 つ
* 表は各観測所について 1 位〜10 位を表示
* 地点名の下に観測開始日を表示

---

### 3. 実況の扱い

実況値は以下のように扱われます。

* ランクインした場合、該当順位に**自動挿入**
* 既存の順位は繰り下げ
* 元の10位は表示外
* 表の右端に実況列を表示（ON/OFF可能）

---

### 4. 色分け

表では主に次の色分けを行います。

* 赤系: 実況でランクインした値
* 黄系: 当年（基準変更可能）の記録
* 赤 + 黄: 両方に該当する値

---

### 5. 実況ランクイン一覧

画面下部に以下を表示します。

* 実況で10位以内に入った項目
* 通年・当月を分けて表示
* クリックすると該当の要素・月へジャンプ

---

### 6. 更新時刻

画面上の更新時刻は、できるだけ「処理完了時刻」ではなく、最後に取得した実況・観測の基準時刻を表示します。

例:

* 10:20 の実況を 10:48 に取得した場合
* 表示は「10:20 時点」

---

### 7. データがない場合

データ未表示のケースは、次のように切り分けて扱います。

* その県・その要素について観測対象がない
* 観測対象はあるが、その回の取得に失敗した
* 県または要素がまだ未対応

---

## データ構造

極値データは以下の形式で保存されています。

```text
data/<region>/<pref>/<element>-<month>.json
```

例:

```text
data/kinki/nara/dailyPrecip-all.json
```

また、全体の更新情報は以下にまとめられます。

```text
data/manifest.json
```

## 地域区分

本アプリでは以下の地域区分を使用しています。

北海道 → hokkaido
東北 → tohoku
関東甲信 → kanto_koshin
北陸 → hokuriku
東海 → tokai
近畿 → kinki
中国 → chugoku
四国 → shikoku
九州北部 → kyushu_north
九州南部・奄美 → kyushu_south
沖縄 → okinawa

※ prefectures.json / Pythonスクリプト / フロント側で完全一致させる必要があります。

---

## データ更新の流れ

このリポジトリでは、GitHub Actions と Python スクリプトを使ってデータを更新しています。

### 極値ランキング

* 6時間ごとに更新

### 実況データ

* 10分ごとに取得（フロント側で処理）

## 補足・注意事項

* 気象庁の仕様上、ランキングが存在しない場合やデータが欠ける場合があります（正常挙動）
* 観測要素は観測所ごとに異なるため、全ての要素が表示されるとは限りません
* 一部の実況データは取得失敗時に直近値を使用する場合があります
* data/live は旧構造の名残であり、現在は使用していません（将来的に削除予定）

# weather_extreme

気象庁の公開データを参考に、都道府県ごとの県内極値一覧を表示する GitHub Pages 用のウェブページです。

## 概要

このページでは、都道府県ごとに各観測所の極値順位を一覧表示します。

表示対象の主な要素は以下のとおりです。

- 降水
  - 日降水量
  - 日最大10分間降水量
  - 日最大1時間降水量
  - 月最大3/6/12/24/48/72時間降水量
  - 月降水量の多い方 / 少ない方
- 気温
  - 日最高気温の高い方 / 低い方
  - 日最低気温の高い方 / 低い方
  - 月平均気温の高い方 / 低い方
- 湿度
  - 日最小相対湿度
- 風
  - 日最大風速
  - 日最大瞬間風速
- 日照
  - 月間日照時間の多い方 / 少ない方
- 雪
  - 降雪の深さ日合計
  - 降雪の深さ月合計
  - 月最大3/6/12/24/48/72時間降雪量
  - 月最深積雪の大きい方 / 小さい方

また、実況値から「県内極値の10位以内に入る見込みがある項目」を上部に表示します。

## ページ仕様

### 1. 地域・都道府県・月の切り替え

ページ上部で以下を選択できます。

- 地域
- 都道府県
- 月
  - 通年
  - 1月〜12月

地域を変えると、その地域に属する都道府県一覧へ切り替わります。

### 2. 要素の切り替え

要素選択パネルから表示する要素を切り替えます。

- 一度に表示する要素は1つ
- 表は各観測所について1位〜10位を表示
- 地点名の下に観測開始日を表示

### 3. 色分け

表では次の色分けを行います。

- 赤系: 実況で10位以内に入る見込みのある値
- 黄系: 直近1年以内の記録
- 赤 + 黄: 両方に該当

### 4. 更新時刻

画面の更新時刻は、原則として「データの取得完了時刻」ではなく、**最後に取得した実況・観測の基準時刻**を表示する想定です。

例:
- 10:20 の10分値を 10:48 に取得完了した場合
- 画面表示は 10:20 時点 とする

### 5. データがない場合の扱い

データが表示されない場合は、以下を区別する方針です。

- その県・その要素について、そもそも観測対象がない
  - 空欄または該当なしとして扱う
- 観測対象はあるが、その回の取得に失敗した
  - エラーメッセージを表示する
- 県自体が未対応
  - 未対応として表示する

### 6. 診断情報

保守しやすくするため、ページ内に診断情報欄を設けます。

診断情報では主に以下を確認できます。

- 選択中の地域・都道府県・月・要素
- 読み込み対象の JSON パス
- manifest の基準時刻
- 実況一覧の取得状況
- 表データの取得状況
- エラー内容
